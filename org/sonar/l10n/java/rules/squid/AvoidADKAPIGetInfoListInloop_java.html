PLM rule to avoid ADK method GetInfoList in Loop

Noncompliant Code for getinfo Example


for (int j = 0; j < theSims.size(); j++) {
	
	Map theNextMap = (Map)theSims.get(j);
	//Get the Allocations for the next Route
	DomainObject theNextRouteDO = DomainObject.newInstance(context, (String)theNextMap.get("id"));
	StringList theNextARAllocationTEIds = theNextRouteDO.getInfoList(context, "to[AR_Allocation].from.id");
}
above getInfo can be placed outside the loop.

Compliant Solution for getinfoList

Sample code snippet would look like this.

StringList theNextARAllocationTEIds ;
 
 for (int j = 0; j < theSims.size(); j++) {

	Map theNextMap = (Map)theSims.get(j);
	//Get the Allocations for the next Route
	DomainObject theNextRouteDO = DomainObject.newInstance(context, (String)theNextMap.get("id"));
	theNextARAllocationTEIds = theNextRouteDO.getInfoList(context, "to[AR_Allocation].from.id");
}
