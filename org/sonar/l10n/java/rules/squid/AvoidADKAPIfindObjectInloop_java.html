PLM rule to avoid ADK method findObject in Loop

Noncompliant Code for findObject Example


for (int j = 0; j < theSims.size(); j++) {
	
	Map theNextMap = (Map)theSims.get(j);
	findObject();
}
above findObject can be placed outside the loop.

Compliant Solution for findObject

Sample code snippet would look like this.

StringList theNextARAllocationTEIds ;
 findObject();
 
 for (int j = 0; j < theSims.size(); j++) {

	Map theNextMap = (Map)theSims.get(j);
	//Get the Allocations for the next Route
}
