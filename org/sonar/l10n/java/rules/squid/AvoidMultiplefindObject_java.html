PLM rule to avoid method multiple findObject

Non compliant Code for findObject Example


MapList templateMapList = DomainObject.findObjects(context, "Route Template", sTemplateName, "*", "*", "*", "",false,objSelects);
MapList simList = DomainObject.findObjects(context, "Simulation", "*", "(!to[AR_Analysis Request])", new StringList("id"));

above findObject can be clubbed into a single DB call as shown below.

Compliant Solution for findObject

Sample code snippet would look like this.
MapList simList = DomainObject.findObjects(context, "Simulation,Route Template", "*", "(!to[AR_Analysis Request])", new StringList("id"));
