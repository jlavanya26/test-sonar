PLM rule to avoid method multiple getInfo

Non compliant Code for getInfo Example


MapList templateMapList = DomainObject.getInfo(context, "Route Template", sTemplateName, "*", "*", "*", "",false,objSelects);
MapList simList = DomainObject.getInfo(context, "Simulation", "*", "(!to[AR_Analysis Request])", new StringList("id"));

above getInfo can be clubbed into a single DB call as shown below.

Compliant Solution for getInfo

Sample code snippet would look like this.
MapList simList = DomainObject.getInfo(context, "Simulation,Route Template", "*", "(!to[AR_Analysis Request])", new StringList("id"));
