PLM rule to avoid “.value” notation as part of an attribute selection

Noncompliant Code for .value Example


StringList selects = new StringList (3);
selects.add(“attribute[“ + attr1 + “].value”);
selects.add(“attribute[“ + attr2 + “].value”);
selects.add(“attribute[“ + attr3 + “].value”); 

above .value can be ignored.

Compliant Solution for get info

Sample code snippet would look like this.

StringList selects = new StringList (3);
selects.add(“attribute[“ + attr1 + “]”);
selects.add(“attribute[“ + attr2 + “]”);
selects.add(“attribute[“ + attr3 + “]”);
