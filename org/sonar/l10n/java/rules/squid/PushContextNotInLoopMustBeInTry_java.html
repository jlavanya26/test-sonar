PLM rule to avoid push context

Noncompliant Code for get info Example


DomainObject dObj = new DomainObject(objectId);
String  sFirstName =(String)dObj.getInfo(context, SELECT_ATTRIBUTE_FIRSTNAME);
String  sLastName =(String)dObj.getInfo(context, SELECT_ATTRIBUTE_LASTNAME); 

above getInfo can be clubbed into a single DB call as shown below.

Compliant Solution for get info

Sample code snippet would look like this.

StringList objectSelectList = new StringList();
objectSelectList.add(SELECT_ATTRIBUTE_FIRSTNAME);
objectSelectList.add(SELECT_ATTRIBUTE_LASTNAME);
DomainObject dObj = new DomainObject(objectId);
Map resultMap = dObj.getInfo(context, objectSelectList);
String  sFirstName = resultMap.get(SELECT_ATTRIBUTE_FIRSTNAME);
String  sLastName = resultMap.get(SELECT_ATTRIBUTE_LASTNAME);
